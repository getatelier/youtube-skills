# Architecture Decision Records (ADRs)

This directory captures significant architectural decisions, their context, and consequences.

## What is an ADR?

An ADR is a short document (1-2 pages) that records an important architectural decision, why it was made, and what trade-offs it entails. ADRs are immutable once accepted; if a decision changes, a new ADR supersedes the old one.

## When to Write an ADR

- Adding a new dependency or tool
- Changing the skill architecture (e.g., new sub-skill category)
- Modifying the execution script pattern
- Changing how references or templates are loaded

## Template

Use `000-template.md` as the starting point. Name files sequentially: `001-short-title.md`.
